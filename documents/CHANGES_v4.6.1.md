## Observation 2 — Emerging risks unchanged for 3 weeks (mostly by design, but worth a check)

Two layers here:
- **The list itself** (Private Credit, CRE, Inflation, China, Geopolitics) is a **curated standing watch-list** — intentionally stable. These are structural risks that persist for quarters; the spine is deliberately not AI-regenerated ("never invent risk mappings"). So the *titles* being the same for 3 weeks is correct.
- **What *should* refresh weekly** is each risk's probability / impact / trend / one-line note **and** the bank implications — generated by the Saturday weekly cron (`/api/cron/weekly`, `30 0 * * 6`) into `weekly:markets`. If `weekly:markets` is empty or expired, the dashboard falls back to the **static base ratings** with no weekly notes — which would look frozen.

So the thing to check: on the Markets page, do the emerging-risk cards show a **weekly "refreshed" date and the one-line notes** under each risk? 
- If **yes and the date is recent** → working as designed; only the ratings move, and they may genuinely be stable in a calm stretch.
- If **no notes / stale date** → the weekly cron isn't landing. Likely causes: the Saturday cron hasn't fired since deploy, or the weekly job failed (it's Anthropic-forced — check `ANTHROPIC_API_KEY` and the cron logs for `/api/cron/weekly`). Trigger it manually once with `?secret=<CRON_SECRET>` and confirm the notes + date appear.

I didn't change this behavior — flagging it so you can tell which case you're in. If you'd like, I can add a small "as of <date>" stamp on the emerging-risks section so staleness is visible at a glance.

## Observation 3 — Colorful Learn tab (built)

- **Section headers** now carry a colored tick + number: Saved Analyses (green), Saved for Later (purple), Concept Library (blue), Weekly Learning (amber). Done via a new optional `accent` prop on `CollapsibleSection` — other tabs are untouched.
- **Saved cards are color-coded by kind:** CRO Conversation = purple, Editorial = blue, Japan & Asia = amber, Research = green. The kind pill is tinted to match.
- **Research cards also get a source chip**, colored by origin: **Bloomberg = amber**, URL = blue, Screenshot = purple, Pasted = grey. So within Saved Analyses you can tell a Bloomberg analysis from a pasted one at a glance.

Palette reuses the existing tokens (mizuho/steel/elevated/calm) so it stays on-brand rather than garish.
